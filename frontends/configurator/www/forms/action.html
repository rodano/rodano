Action

Documentation

If the action is documentable, the user will be asked to provide a rationale to justify why he is doing the action. It is possible to configure a list of predefined rationales that will be presented to the user (but the user will always be able to provide a custom rationale).

Signature

If the action requires signature, the user will be prompted to enter his login and password to validate the action. It is possible to configure the affidavit that will be shown to the user.
